大致上讲了一些还没有面试时对于面试的准备和看法


概述: 投简历的前一天 对之前的一些总结吧

##准备

##流程介绍

1.进公司前的一些准备
进公司前先整理好仪表,敲门,进门之后先说你好并且将自带的一份纸质简历双手递交并且简历的正面要朝向面试官
2.进入后
然后一般就开始自我介绍了,一般面试官会说请开始自我介绍吧(如果没有,这时候你要谦卑说”我可以开始自我介绍了吗”),在你认为经过他的同意后开始自我介绍
他说不可以的话 就直接走吧……
3.自我介绍
自我介绍没什么好说的,大概就是介绍一些自己的情况,来这里是干嘛的面试什么职位,毕业学校可以不说,扬长避短,他不问你就不说,当然如果985 211当我没说。大概就是介绍一下你会些
什么技能,之前做过一些那些项目以及你的一些兴趣爱好等,能说谢谢就尽量说个谢谢,语速要适当,就像平常聊天一样不然他会以为你是背的,自信的技术可以
适当的声音大点、不懂瞎写的就一笔带过吧,中途可能会打断你进行提问,需做好准备。建议不会的东西不要写,至少要能说

打招呼 -> 个人资料 -> 职位 -> 个人技能 -> 经验 -> 兴趣爱好 -> 结束的谢谢
    = = 大致就是从这几个方面入手  让自己尽量自然点吧 个人觉得面试多了自然就自然了
    如果可以尽量写一些新知识新技术,兴趣爱好就写点团体活动表明你不是一个孤儿,喜欢浏览技术博客表现你多热爱这个行业且求真求美

4.工作经历
写一个在天眼查不到的公司,他问你就说小公司吧 自己家里人开的 所以进去的早
5.项目经验
一般写个4 5 个项目吧 一个项目周期大概 3-5个月 规模可以10到20人不等 尽量写的规模小点的项目 不然容易引起怀疑被细问
项目可以从一些已经开发好的app 官网中大致可以理解一下功能模块 然后稍微百度了解一下业务逻辑 差不多就行了 一些有亮点的技术会被追问要提前做好准备
项目间可以空个几个月 时间短可以说帮之前的公司进行一些后续的需求开发和维护,时间长可以说公司的人才培养去参与了培训
项目类型的话 大概就杜绝OA CRM 超市管理 图书管理这种烂大街的管理项目,找一些符合社会现状比较有热点热度的项目才能引起别人的兴趣
项目介绍的话应该是从这几个方向:
从目的 -> 功能 -> 项目类型 -> 各个功能模块 -> 项目架构和技术亮点 -> 你在其中所使用那些技术做了哪些事
6.一些可能面到的问题
多线程(几乎是必问的,多方打听)、泛型集合、同步问题、HTTP/TCP协议、AJAX、Tomcat优化、数据库的优化以及SSM、SSH框架架构原理实现等这些是必须的
一些比较流行的项目: Redis非关系型数据库、ActiveMQ消息队列、SpringBoot微服务架构快速开发

以上,纯个人理解,希望对各位有所帮助吧,祝各位面试成功,也祝我面试成功!
